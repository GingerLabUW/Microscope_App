PI Python Libraries Documentation

PIPython is a collection of Python modules to access a PI device and process GCS data. It can be used with Python 2.6+ and 3.3+ on Windows, Linux and OS X and over a socket also on any other platform.

Quickstart

Communicate to a PI device via GCSDevice which wraps the GCS DLL functions and provides methods to connect to the device. Call GCSDevice with the controller name as argument.

1 from pipython import GCSDevice
2 gcs = GCSDevice('C-884')
3 gcs.InterfaceSetupDlg()
4 print gcs.qIDN()
5 gcs.CloseConnection()

GCSDevice is a context manager which closes the connection if an exception raises inside the with statement.

1 from pipython import GCSDevice
2 with GCSDevice('C-884') as gcs:
3  gcs.InterfaceSetupDlg()
4  print gcs.qIDN()

See also the provided samples in the samples subdirectory. Start with quickstart.py.

Requirements

There are no dependencies to additional packages.

PIPython has been tested with Python 2.7.2, Python 3.3.5, Python 3.4.3, Python 3.5.0, each as 32 and 64 bit architecture.

All GCS functions passed at least these test cases:

  • Verification of number and length of function arguments.
  • GCS command that is sent to the device.
  • Handling of the answer returned from the device.
  • Proper type(s) and values of the answer returned by the function.

Arguments

From now on gcs refers to a connected GCSDevice instance.

You can call a setter function with

  • a dictionary of axes/channels and values
  • a list for axes/channels and a list of the values
  • a single item for axis/channel and a single value
1 gcs.MOV({'X': 1.23, 'Y': 2.34})
2 gcs.MOV(['X', 'Y'], [1.23, 2.34])
3 gcs.MOV('X', 1.23)

For channels and numeric axis names you can omit the quotes.

1 gcs.MOV({1: 1.23, 2: 2.34})
2 gcs.MOV([1, 2], [1.23, 2.34])
3 gcs.MOV(1, 1.23)

Getter commands can be called with

  • a list of axes/channels
  • a single item for axis/channel, without quotes if numeric
  • without any arguments which will return the answer for all available axes/channels
1 gcs.qPOS(['X', 'Y'])
2 gcs.qPOS('X')
3 gcs.qPOS(1)
4 gcs.qPOS()

Return values

Axes or channel related answers are returned as (ordered) dictionary.

1 gcs.qPOS()
2 >>>{'X': 1.23, 'Y': 2.34}

If you provide arguments their types are preserved and you can use these as keys.

1 pos = gcs.qPOS([1, 2, 3])
2 print pos[1]

If you do not provide arguments you always have to use strings as keys.

1 pos = gcs.qPOS()
2 print pos['1']

The following sample will move all axes to targets and waits until the motion has finished. It shows how to use only the values from the returned dictionary.

1 from time import sleep
2 ...
3 gcs.MOV(axes, targets)
4 while not all(gcs.qONT(axes).values()):
5  sleep(0.1)

Helpers

In pipython.pitools you will find some helper funtions for your convenience. See the provided samples for how to use them. The sample above can then be written as:

1 import pipython.pitools as pi
2 ...
3 gcs.MOV(axes, targets)
4 pi.waitontarget(gcs, axes)

Enable debug logging

To log debug messages on the console just enter these lines prior to calling GCSDevice.

1 from logging import basicConfig, DEBUG
2 basicConfig(level=DEBUG)

GCSError and error check

By default an "ERR?" command is sent after each command to query if an error occurred on the device which then will be raised as GCSError exception. If communication speed is an issue you can disable error checking.

1 gcs.errcheck = False

To handle a catched GCSError exception you can use the defines provided by gcserror instead of pure numeric values. Remember the difference between GCSError which is the exception class and gcserror which is the according module.

1 from pipython import GCSDevice, GCSError, gcserror
2 with GCSDevice('C-884') as gcs:
3  try:
4  gcs.MOV('X', 1.23)
5  except GCSError as exc:
6  if exc == gcserror.E_1024_PI_MOTION_ERROR:
7  print 'There was a motion error, please check the mechanics.'
8  else:
9  raise

The exception class GCSError will translate the error code into a readable message.

1 from pipython import GCSError, gcserror
2 raise GCSError(gcserror.E_1024_PI_MOTION_ERROR)
3 >>>GCSError: Motion error: position error too large, servo is switched off automatically (-1024)

With the property GCSDevice.embederr you can embed the error querying command into a GCS set command. Hence only one package needs to be sent to the device which also can improve communication performance.

Big data

Commands like qDRR() which read a large amount of GCS data return immediately with the header dictionary containing information about the data. Then they will start a background task that carries on reading data from the device into an internal buffer. The bufstate property returns the progress of the reading as floating point number in the range 0 to 1 and turns to True when reading has finished. Hence, when using it in a loop check for is not True. (Remember, this is not the same as != True.)

1 header = gcs.qDRR(1, 1, 8192)
2 while gcs.bufstate is not True:
3  print('read data {:.1f}%...'.format(gcs.bufstate * 100))
4  sleep(0.1)
5 data = gcs.bufdata

Textual interface

Besides the functions implemented in GCSCommands you can send GCS commands as strings to the controller. Use ReadGCSCommand() for commands returning an answer, ReadGCSData() for commands returning GCS data and GcsCommandset() for non-answering commands.

1 print gcs.ReadGCSCommand('POS?')
2 print gcs.ReadGCSData('DRR? 1 100 1')
3 gcs.GcsCommandset('MOV X 1.23')

It returns the raw string or GCS data from the controller. If errorcheck is enabled the error state is queried from the device automatically. We recommend to use the provided functions instead of sending raw strings.
